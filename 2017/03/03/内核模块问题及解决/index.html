内核模块问题及解决

发表于 2017-03-03 | | 阅读次数

针对近期遇到的需求，记录解决方案以及对问题的分析，根据深入学习不断对本文补充完善。

内核文件读写

内核空间无法直接使用系统调用read、write函数

使用vfs_read vfs_write

mm_segment_t fs = get_fs();
set_fs(KERNEL_FS);
//vfs_write();
vfs_read();
set_fs(fs);

打开文件 使用绝对路径

fp =filp_open("/home/file",O_RDWR | O_CREAT,0644);
   if (IS_ERR(fp)){
       printk("create file error/n");
       return -1;
   }

oops
insmod 模块后无法写入和读取文件中的数据，通过加内核打印，屏蔽函数等方法，定位问题在vfs上。
vfs读写文件的参数要使用已filp_open返回的文件指针，而非文件的绝对路径，使用后需要close。

注意空指针，大量kernel panic，oops都是由于空指针造成的，如

内核模块参数传递

需要包含头文件：
模块文件 main.c 中添加全局变量，通过宏 module_param(name, type, perm)定义一个模块参数。通过module_param_string(name, string, len, perm)传递数组
name是外部参数名，string是内部变量名，len是以string命名的buffer大小，perm表示sysfs的访问权限。示例：

static char *name ;
static char words[BUF_LEN];
module_param(name, charp, S_IRUGO);
module_param_string(words,words,BUF_LEN, S_IRUGO);

insmod 模块时传入参数

$ sudo insmod XXX.ko words=TestString name=TestName

内核模块无法卸载问题：

场景复现：
insmod文件系统wrapfs模块，挂载文件系统多次，直接rmmod wrapfs

$sudo insmod wrapfs.ko
$mount -t wrapfs /path/to/device /path/to/dir

卸载时出错

rmmod: ERROR: Module wrapfs is in use

查看 lsmod | grep wrapfs 发现引用计数为 1
umount 对应类型的文件系统

$sudo umount /path/to/device
